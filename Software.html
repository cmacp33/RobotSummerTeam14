Software Overview

To upload our code we flashed the ESP32 from VS code using platformIO. Most of the uploads were done over microUSB however we experimented with using OTA (wireless updates using wifi) for tuning and testing. We used esp_now.h and Wifi.h libraries for MCU communication and the ESP32Servo.h library for controlling our servo motors. Additionally we used FreeRTOS for task management and scheduleing so we could run multiple functions concurrently.

MCU Communication

To be successful our robots needed to communicate with one another to correctly time their actions. We used ESP-NOW wifi as our communication protocal. The communication process works as follows:

1. Robot 1 waits behind counter
2. Robot 2 carries out the required action
3. Robot 2 sends a boolean that tells robot 1 to resume action
4. Robot 1 returns to its function stack
