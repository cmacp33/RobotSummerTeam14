Development Platform

To upload our code we flashed the ESP32 from VS code using platformIO. Most of the uploads were done over microUSB however we experimented with using OTA (wireless updates using wifi) for tuning and testing.

Task Handling

To increase efficieny we structured our software around freeRTOS task handling. free RTOS allowed us to run multiple functions concurrently meaning we could make claw adjustments and move around at the same time. It also kept our code organized and made testing much easier.

MCU Communication

To be successful our robots needed to communicate with one another to correctly time their actions. We used ESP-NOW wifi as our communication protocal. The communication process works as follows:
1. Robot 1 stops at the counter and gets stuck in an infinite loop
2. Robot 2 carries out the required action
3. Robot 2 sends a boolean flag that tells robot 1
4. The flag breaks robot 1 out of the infinite loop and it returns to its function stack
